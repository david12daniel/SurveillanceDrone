### 6. Onboard Recording Architecture Decision (Phase 3)
- **Decision:** Phase 3 will store thermal footage onboard via an inline DVR, not rely solely on live downlink
- Calculated storage needs for 30-min flight: Lepton 3.5 raw = ~500 MB, Boson 640 compressed = ~4.4 GB
- Entry-level 32 GB microSD ($8-12) holds many flights — storage is not a constraint
- This eliminates need for second VTX for thermal in Phase 3
- **ThermalVideoRecorder** (inline DVR) added to systems_engineering_plan.md Phase 3 components and to model.md as a new `part def`

## Design Decisions Made

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Camera architecture | **Dual (FPV + Thermal)** | Need FPV cam for piloting, thermal for mission — thermal too low-res for flight |
| Thermal downlink (Phase 3) | **Onboard recording via inline DVR** | Eliminates second VTX, survives signal loss, ~$20-30 part, negligible weight |
| Telemetry approach | **ELRS over single RF link** | Proven 3.5+ km range, bidirectional, saves cost/weight/UART vs separate telemetry radio |
| TelemetryTransmitter | **Merged into RadioReceiver** | ELRS RX handles both RC control and telemetry return — no separate component needed |
| KOLAS7 options kept | **PNP, BNF Analog, BNF HD** | Frame kit dropped per David's direction |
| GPS priority | **Critical for Phase 2 (waypoints)** | Budget +$25 if airframe doesn't include one |
